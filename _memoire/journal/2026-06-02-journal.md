# Journal — 2026-06-02

## Sujet : Naissance du produit SaaS « Boutique IA » (étage 1)

### Contexte / décision stratégique
Mongazi veut un produit digital scalable (vision « le million ») : un **vendeur WhatsApp IA self-service** pour commerçants d'Afrique de l'Ouest. Après analyse marché, c'est le produit n°1 retenu.

Architecture en **4 étages**, construits en vagues (comme NOVA) :
1. Inscription self-service + paiement Mobile Money + alertes — **FAIT, déployé**
2. Cerveau IA WhatsApp (multi-boutiques) — à coder
3. Vente complète (commande + encaissement) — à coder
4. Tableau de bord admin — à coder

### Décisions techniques
- **PAS de n8n** : logique en code Python/FastAPI (comme NOVA).
- **PAS de FedaPay au départ** : Mobile Money manuel (client envoie au numéro + capture, patron valide). FedaPay rebranché plus tard.
- **Option A retenue** pour WhatsApp : NEBULA fournit le numéro (clé en main).
- Numéros de production : à trancher plus tard (Twilio = CB requise ; Meta Cloud API = 1000 conv/mois gratuites, mieux adapté). Étage 2 sera d'abord testé sur le **WhatsApp Sandbox gratuit** (sans carte).

## Étage 3 — Vente complète ✅ (commande + alerte commerçant)

### Principe
Le vendeur IA dispose d'un **outil `enregistrer_commande`** (tool-use Claude). Dès que le
client confirme (produit + quantité + livraison/retrait + adresse si livraison), l'agent
appelle l'outil **avant** de donner les instructions de paiement → la commande est écrite
dans `bia_orders` et le commerçant est prévenu.

### Réalisations
- `core/brain.py` : définition `ORDER_TOOL` + boucle tool-use (max 4 tours, `did_tool`,
  fallback forcé si l'agent termine sans message client). Section commande ajoutée au prompt.
- `db/client.py` : `create_order()` (insert `bia_orders`) + `list_orders()` (prêt étage 4).
- `notify.py` : `notify_new_order()` → alerte Telegram à Mongazi (canal actif) **+**
  WhatsApp direct au patron via API Twilio (`_whatsapp_owner`, `_to_e164`) — **dormant**
  tant que `twilio_account_sid` / `twilio_auth_token` absents du `.env`.
- `web/server.py` : `_order_recorder()` branché sur le webhook WhatsApp (vrais clients) et
  `/api/chat`. L'essai gratuit (preview) **n'enregistre pas** de commande.
- `config.py` : champs `twilio_account_sid`, `twilio_auth_token`.
