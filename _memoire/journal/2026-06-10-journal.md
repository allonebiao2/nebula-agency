## 10. METERING CONVERSATIONS + CRÉDITS (grille précise + soft cap) — déployé
Décision : packager la valeur + monétiser le dépassement SANS jamais couper le vendeur (sinon ventes perdues = churn).
- **Grille (config.py, ajustable — financier = Mongazi)** : `PLAN_CONV_INCLUDED` = Démarrage **300**/mois · Business **1200** · Empire **illimité**. `CREDIT_PACKS` (prépayé MoMo, dégressif) : 100=2000F · 300=5000F · 1000=12000F (le petit pack > tarif forfait → pousse à l'upgrade). Helper `conv_included_for_plan`.
- **Métrique = conversations/MOIS** (clients distincts ayant écrit ce mois), PAS messages, PAS journalier (qui tuerait les jours de pointe). `db.conversations_used_this_month` (distinct customer_whatsapp depuis début de mois WAT) + `conversation_usage(merchant)` → {used, included, credits, allowance, remaining, pct, status: unlimited/ok/warning≥80%/over}. Colonne `bia_merchants.conv_credits` (migration `add_conversation_credits`). `add_conversation_credits(id, n)`.
- **SOFT CAP** : purement informatif + nudge. **Le vendeur n'est JAMAIS bloqué** (aucun hard stop ajouté). Cadrage POSITIF (« votre agent a géré X conversations 💪 »), le nudge recharge/upgrade n'apparaît qu'à ≥80%.
- **Back-office** : section « Vos conversations ce mois » (barre de progression, packs + CTA à ≥80%/dépassement). **Cockpit admin** : usage X/allowance + crédits par boutique (repère les gros = upsell) + bouton **+Crédits** (validation manuelle d'une recharge MoMo) → endpoint `POST /api/admin/merchants/{id}/credits` (pack id ou nombre, négatif pour corriger).
- **Assistant perso** : nouvel outil `etat_forfait` (« où j'en suis ? », « combien de conversations ce mois ? ») = même métrique réelle → devient le canal d'upsell naturel (seulement si on demande).
- **Écarté (dangereux)** : hard cap/coupure du vendeur, débit auto d'argent, complexité de settlement mensuel des crédits (risque de bug facturation). Crédits = solde persistant géré par l'admin (v1). Limites larges + invisibles avant 80% (ne pas effrayer les early adopters).
- Tests : compile + metering contre vraie base (usage réel, crédits +/−, outil) + Jinja 2 templates OK. Déployé sain (endpoint credits 401).
