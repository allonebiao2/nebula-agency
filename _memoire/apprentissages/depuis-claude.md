## Apprentissages

### 2026-05-11 — Comment structurer une mémoire projet pour Claude Code

**Question / contexte** : Comment éviter de tout réexpliquer à Claude Code à chaque session.

**Ce que Claude a expliqué** :
- Un `CLAUDE.md` à la racine sert de cerveau projet, chargé automatiquement
- Compléter avec un dossier `_memoire/` (long-terme) et `_knowledge/` (spécialisé)
- Templates dans `_templates/` pour répliquer le même standard à chaque nouveau client
- Logger les sessions importantes dans `_memoire/conversations/`

**Ce que j'en retiens / vais appliquer** :
- Tout passe par CLAUDE.md → fichiers spécialisés en lecture à la demande
- Un client = un dossier, toujours la même forme
- Ne pas mélanger long-terme (mémoire) et court-terme (tâches en cours)

**Source / lien** : Session Claude Code du 2026-05-11
