# Journal — 2026-06-02

## Sujet : Naissance du produit SaaS « Boutique IA » (étage 1)

### Contexte / décision stratégique
Mongazi veut un produit digital scalable (vision « le million ») : un **vendeur WhatsApp IA self-service** pour commerçants d'Afrique de l'Ouest. Après analyse marché, c'est le produit n°1 retenu.

Architecture en **4 étages**, construits en vagues (comme NOVA) :
1. Inscription self-service + paiement Mobile Money + alertes — **FAIT, déployé**
2. Cerveau IA WhatsApp (multi-boutiques) — à coder
3. Vente complète (commande + encaissement) — à coder
4. Tableau de bord admin — à coder

### Décisions techniques
- **PAS de n8n** : logique en code Python/FastAPI (comme NOVA).
- **PAS de FedaPay au départ** : Mobile Money manuel (client envoie au numéro + capture, patron valide). FedaPay rebranché plus tard.
- **Option A retenue** pour WhatsApp : NEBULA fournit le numéro (clé en main).
- Numéros de production : à trancher plus tard (Twilio = CB requise ; Meta Cloud API = 1000 conv/mois gratuites, mieux adapté). Étage 2 sera d'abord testé sur le **WhatsApp Sandbox gratuit** (sans carte).

## Étage 2 — Le cerveau IA + rebrand Vendora

### Cerveau IA (`core/brain.py`)
- Construit un system prompt à partir de la fiche boutique (produits, prix, ton, livraison, paiement MoMo).
- Prompt caching (cache_control ephemeral) sur la fiche → tokens économisés.
- Playbook de vente intégré (bénéfices, upsell, closing, objections, preuve sociale).
- Style : réponses TRÈS courtes, format WhatsApp (un seul `*`, plus de `**`), max_tokens=400.
- Mémoire de conversation via `bia_messages` (load_history / save_message).

### Démo + essai
- Simulateur chat `/demo/{id}` (interface type WhatsApp).
- Essai gratuit `/essai/{id}` après inscription : `free_trial_messages=6`, limite côté serveur (preview), puis CTA activer.
- Souscription directe possible (bouton Activer toujours visible).
- Inscription redirige désormais vers `/essai/{id}`.
