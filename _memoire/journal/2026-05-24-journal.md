## Session 9 — Photos produits : retour aux originaux fond blanc + normalisation

### Demande Gloria
Les fonds blancs des produits sur la vitrine ont été très mal enlevés (transparence
PNG ratée → halos, crops trop serrés, perte de qualité). Remettre les images avec
leurs fonds normaux. Tous les produits doivent être à la même échelle, même

### Diagnostic
- Les originaux propres (JPEG, fond blanc studio) étaient préservés dans
  `clients/04-luxury-skin-clinic/assets/images/...`.
- Les versions embarquées en base64 dans `ina-luxury.html` étaient des PNG
  transparents lourds (objet IMG = **11.7 Mo** chars, soit ~8 Mo binaire).
- Le fichier `ina-luxury.html` faisait **12 Mo** au total → lourd à charger, à éditer.

### Pipeline de normalisation
Script PowerShell GDI+ (`$env:TEMP\process-ina-imgs.ps1` et `process-cozy-imgs.ps1`) :
1. Lecture du JPEG original.
2. Création d'un canvas **600 × 800 px (ratio 3:4) blanc pur**.
3. Image source dimensionnée pour **rentrer dans le canvas** (preserve ratio), centrée.
4. Marges blanches automatiques quand le produit est plus carré/portrait.
5. Sauvegarde en JPEG qualité 78.
6. Encodage base64 → injection dans la ligne `const IMG={...}`.
